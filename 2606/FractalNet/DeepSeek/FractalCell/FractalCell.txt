🔍 В чем сейчас главная боль (Архитектурный инсайт)
Посмотрите, что у нас есть:
У вас есть шикарные шаблоны BehaviorTemplate и EventBehaviorTemplate<T>.
У вас есть FractalCellFactory, который создает ячейку и вызывает behavior.AttachToAsync(cell).
НО! В самом FractalCellTemplate (в методе ExecuteAsync или ConfigureHandlers) нет кода, который передавал бы события из шины в прикрепленные поведения!
Сейчас поведение «пристегивается» к ячейке, но ячейка о нем «не знает». Событие бегает по InternalBus, обрабатывается хардкодом в ChannelFractalCell (или TplFractalCell), а до HeartbeatBehavior или DataProcessingBehavior оно фактически не доходит автоматически. Вам приходится вручную продумывать маршрутизацию, именно поэтому это «дается не просто».
🎯 Как сделать добавление поведений ЛЕГКИМ
Чтобы освоение механизма поведений пошло как по маслу, нам нужно внедрить в FractalCellCore концепцию BehaviorRouter (Маршрутизатор поведений).
Это небольшая, но критически важная структура внутри FractalCellTemplate.
Как это будет работать (Идеальный сценарий):
Фабрика создает ячейку и аттачит к ней поведение.
Ячейка сохраняет его во внутренний список: protected List<IBehavior> _behaviors = new();.
Когда в InternalBus (или прямо во FractalCellTemplate.ExecuteAsync) прилетает событие, ячейка автоматически пробегает по всем _behaviors и спрашивает:

🗺️ План «Освоения Поведений» (Roadmap)
Предлагаю не пытаться охватить всё сразу, а идти плавно, как вы и сказали. Вот 3 ступеньки, которые снимут 90% сложности:
Ступень 1: Автоматический Роутер (BehaviorRouter)
Задача: Научить FractalCellTemplate автоматически отдавать события поведениям.
Результат: Поведения начинают реально работать и получать ивенты.
Ступень 2: Жизненный цикл (Lifecycle Hooks)
Задача: Сделать так, чтобы при вызове cell.StartAsync() ядро само обзвонило все поведения и вызвало у них OnCellStartedAsync().
Результат: Поведения смогут инициализировать свои ресурсы (например, подключаться к БД или запускать таймеры) без костылей.
Ступень 3: Композитные ячейки (Множественные поведения)
Задача: Научить ячейку нести на себе сразу несколько поведений (например, Heartbeat + DataProcessing + ErrorLogger), которые работают как конвейер.
Что скажете? Начнем со Ступени 1 (Автоматический Роутер)? Мы внесем небольшие правки в FractalCellTemplate и FractalCellFactory (в Core), и вы сразу почувствуете, как сопротивление уйдет, а добавление новых фич в FractalCellApp станет приносить удовольствие. 🚀

Превосходно! 🚀 Внедрение Автоматического Роутера — это тот самый "переломный момент", после которого работать с вашим фреймворком станет невероятно приятно.
Мы сделаем так, что FractalCellTemplate вообще не будет знать о том, как обрабатываются события. Она просто предоставит свою InternalBus, а поведения сами будут к ней подключаться в момент прикрепления. Это называется Inversion of Control (IoC) и паттерн Observer.

// ... внутри класса FractalCellTemplate ...

protected readonly List<IBehavior> _behaviors = new();
public IReadOnlyList<IBehavior> Behaviors => _behaviors.AsReadOnly();

public void AddBehavior(IBehavior behavior)
{
    if (!_behaviors.Contains(behavior))
    {
        _behaviors.Add(behavior);
    }
}

public override async Task StartAsync(CancellationToken cancellationToken)
{
    Logger.LogInformation("FractalCell {CellId} starting", Configuration.CellId);

    // 🆕 1. Lifecycle: OnCellStartingAsync
    foreach (var behavior in _behaviors.OfType<ILifecycleBehavior>())
        await behavior.OnCellStartingAsync(cancellationToken);

    foreach (var bs in CreateBackgroundServices())
    {
        _backgroundServices.Add(bs);
        await bs.StartAsync(cancellationToken);
    }

    await InternalBusField.StartAsync(cancellationToken);
    _ = ExecuteAsync(cancellationToken);

    // 🆕 2. Lifecycle: OnCellStartedAsync
    foreach (var behavior in _behaviors.OfType<ILifecycleBehavior>())
        await behavior.OnCellStartedAsync(cancellationToken);

    Logger.LogInformation("FractalCell {CellId} started successfully", Configuration.CellId);
}

public override async Task StopAsync(CancellationToken cancellationToken)
{
    Logger.LogInformation("FractalCell {CellId} stopping", Configuration.CellId);

    // 🆕 3. Lifecycle: OnCellStoppingAsync
    foreach (var behavior in _behaviors.OfType<ILifecycleBehavior>())
        await behavior.OnCellStoppingAsync(cancellationToken);

    await InternalBusField.StopAsync();

    foreach (var bs in _backgroundServices)
        await bs.StopAsync(cancellationToken);

    // 🆕 4. Lifecycle: OnCellStoppedAsync
    foreach (var behavior in _behaviors.OfType<ILifecycleBehavior>())
        await behavior.OnCellStoppedAsync(cancellationToken);

    Logger.LogInformation("FractalCell {CellId} stopped", Configuration.CellId);
}

Шаг 3. 🔥 МАГИЯ: Автоматическая подписка в EventBehaviorTemplate
Это сердце нашего роутера. Когда поведение прикрепляется к ячейке (AttachToAsync), оно само берет InternalBus ячейки и подписывается на нужный тип событий. Ядро при этом остается абсолютно чистым!
📄 Core/Templates/BehaviorTemplate.cs
(Замените класс EventBehaviorTemplate<TEvent> на этот)
public abstract class EventBehaviorTemplate<TEvent> : BehaviorTemplate, IEventHandlingBehavior
    where TEvent : IApplicationEvent
{
    private IDisposable? _subscription; // 🆕 Храним подписку, чтобы потом отписаться

    protected EventBehaviorTemplate(ILogger? logger = null) : base(logger) { }

    // 🆕 АВТОМАТИЧЕСКИЙ РОУТЕР: Срабатывает сразу после AttachToAsync
    protected override Task OnAttachedAsync(CancellationToken ct)
    {
        if (_attachedCell?.InternalBus != null)
        {
            _subscription = _attachedCell.InternalBus.Subscribe<TEvent>(async @event =>
            {
                try
                {
                    if (await CanHandleAsync(@event))
                    {
                        await HandleEventAsync(@event);
                    }
                }
                catch (Exception ex)
                {
                    _logger?.LogError(ex, "Error in behavior {BehaviorId}", BehaviorId);
                    if (this is IErrorHandlingBehavior errHandler)
                        await errHandler.OnErrorAsync(ex, @event);
                }
            });
            
            _logger?.LogInformation("🔌 Behavior {BehaviorId} auto-subscribed to {EventType} on cell {CellId}",
                BehaviorId, typeof(TEvent).Name, _attachedCell.CellId);
        }
        return base.OnAttachedAsync(ct);
    }

    // 🆕 Чистим за собой при отвязке
    protected override Task OnDetachedAsync(CancellationToken ct)
    {
        _subscription?.Dispose();
        _subscription = null;
        return base.OnDetachedAsync(ct);
    }

    public virtual Task<bool> CanHandleAsync(IApplicationEvent @event)
        => Task.FromResult(@event is TEvent);

    public virtual async Task HandleAsync(IApplicationEvent @event)
    {
        if (@event is TEvent typedEvent)
            await HandleEventAsync(typedEvent);
    }

    protected abstract Task HandleEventAsync(TEvent @event);
}
Шаг 4. Обновляем FractalCellFactory (Связываем всё вместе)
Фабрика теперь должна сначала "зарегистрировать" поведение в ячейке (чтобы сработали Lifecycle-хуки), а затем "прикрепить" его (чтобы сработала авто-подписка на шину).
📄 Core/FractalCellFactory.cs
(Обновите методы CreateWithBehaviorAsync и CreateWithBehaviorsAsync)
public static async Task<IFractalCell> CreateWithBehaviorAsync<TBehavior>(
    FractalCellConfiguration config,
    IFractalEventHub hub,
    ILoggerFactory loggerFactory,
    object? behaviorConfig = null,
    CancellationToken ct = default)
    where TBehavior : IBehavior, new()
{
    var cell = await CreateAsync(config, hub, loggerFactory, ct);
    var behavior = new TBehavior();

    cell.AddBehavior(behavior);             // 🆕 Для Lifecycle
    await behavior.AttachToAsync(cell, ct); // 🆕 Для Авто-подписки

    if (behaviorConfig != null)
        await behavior.ConfigureAsync(behaviorConfig, ct);

    return cell;
}

public static async Task<IFractalCell> CreateWithBehaviorsAsync(
    FractalCellConfiguration config,
    IFractalEventHub hub,
    ILoggerFactory loggerFactory,
    IEnumerable<IBehavior> behaviors,
    CancellationToken ct = default)
{
    var cell = await CreateAsync(config, hub, loggerFactory, ct);

    foreach (var behavior in behaviors)
    {
        cell.AddBehavior(behavior);             // 🆕
        await behavior.AttachToAsync(cell, ct); // 🆕
    }

    return cell;
}
Шаг 5. Важная уборка (Удаление хардкода)
В вашем текущем коде в ChannelFractalCell.cs и TplFractalCell.cs есть внутренние воркеры (ChannelBackgroundService и TplBackgroundService), внутри которых захардкожен switch (@event.EventType) (обработка Heartbeat и ProcessData).
Теперь они стали не нужны и даже вредны! 🛑
Если вы их оставите, они будут "перехватывать" события и конкурировать с вашими новыми HeartbeatBehavior и DataProcessingBehavior.
Что нужно сделать:
В ChannelFractalCell.ConfigureHandlers() и TplFractalCell.ConfigureHandlers() оставьте только системный роутинг (проверку TargetCellId и форвардинг в ExternalBus).
Удалите или полностью очистите методы ProcessEventAsync (в Channels) и ProcessInWorkerAsync (в TplDataflow), где был switch. Вся бизнес-логика теперь живет исключительно в FractalCellApp.Behaviors!
