Разбор ключевых моментов реализации
1. Зависимость от абстракций
Класс ReportService принимает в конструкторе интерфейс IReportStrategy. 
Внутри метода CreateReport() он вызывает _strategy.Generate(data), даже не зная, 
превратятся ли данные в строку с фигурными скобками (JSON) или в строки с запятыми (CSV). Это принцип инверсии зависимостей (DIP).

2. Взаимозаменяемость во время выполнения
В методе Main мы создаем экземпляры сервисов:

Сначала new ReportService(new CsvReportStrategy()).
Затем new ReportService(new JsonReportStrategy()).
Объект reportService остается тем же типом (ReportService), 
но его внутреннее поведение кардинально меняется благодаря подмене объекта-стратегии.

3. Соблюдение принципа OCP (Open/Closed)
Если завтра бизнесу понадобится экспорт в PDF или XML, вам не придется открывать файл ReportService.cs. 
Вы просто создадите новый класс:

csharp
Копировать
public class PdfReportStrategy : IReportStrategy
{
    public string Generate(IEnumerable<DataItem> data) { /* логика */ }
}
и передадите его в конструктор сервиса.

4. Обработка специфики форматов
Обратите внимание на реализацию CsvReportStrategy: там добавлена проверка на наличие запятых в поле Name. 
Если название содержит запятую (например, "Смартфон, флагман"), оно оборачивается в кавычки, 
чтобы структура CSV-файла не сломалась. 
Стратегия инкапсулирует в себе все нюансы своего формата.
